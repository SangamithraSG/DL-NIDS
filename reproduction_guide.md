# DL-NIDS Reproduction & Verification Guide

This guide describes the exact steps to reproduce the system's results from scratch.

## 1. Environment Setup
Ensure all dependencies are installed:
```bash
pip install -r requirements.txt
```

## 2. End-to-End Workflow

### Step 1: Preprocessing
Clean, encode, and scale the NSL-KDD dataset. Generate sliding-window sequences for deep learning.
```bash
python main.py preprocess --multiclass
```
*   **Artifacts**: `artifacts/scaler.joblib`, `artifacts/encoder.joblib`.

### Step 2: Training
Train the models in sequence.
```bash
# Unsupervised Anomaly Detection
python main.py train --model autoencoder

# Sequential Deep Learning
python main.py train --model lstm
python main.py train --model hybrid

# baseline ML
python main.py train --model rf
```
*   **Checkpoints**: `saved_models/*.pt` and `saved_models/random_forest.joblib`.

### Step 3: Evaluation
Verify performance metrics on the test set.
```bash
python main.py evaluate --model autoencoder
python main.py evaluate --model lstm
python main.py evaluate --model hybrid
python main.py evaluate --model ensemble
```

### Step 4: Analysis Dashboard
Launch the interactive dashboard.
```bash
streamlit run dashboard/app.py
```

## 3. Storage Locations
- **Raw Data**: `data/KDDTrain+.txt`, `data/KDDTest+.txt`
- **Logged History**: `logs/` (CSVs for every training run)
- **Model Weights**: `saved_models/`
- **Visual Reports**: `reports/` (Generated by dashboard and Stage 5 scripts)
- **Preprocessing state**: `artifacts/`

## 4. Verification Checklist
- [ ] `logs/` contains `autoencoder_history.csv`, `lstm_history.csv`, etc.
- [ ] `saved_models/` contains weights for 4 models.
- [ ] `main.py evaluate --model ensemble` returns an FPR below 4%.
